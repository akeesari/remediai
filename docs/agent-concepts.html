2. LangGraph Orchestration
Graph-based engine drives the agent pipeline
LangGraph models the 6-agent pipeline as a StateGraph. Each node is a Python async function; typed edges enforce execution order. A conditional edge at Bug Creation either ends the MVP loop or routes to the Phase 2 approval branch.
✓
Typed IncidentState flows through every node unchanged
✓
Conditional edge: MVP end vs. Phase 2 approval branch
✓
Checkpointing allows pipeline resume after transient failure
✓
Full agent_trace JSON written to audit_log after each run

3. Agentic RAG
Hybrid retrieval grounds every fix recommendation
The RAG Retrieval Agent queries Azure AI Search with BM25 keyword and vector search in one call. Results are re-ranked by document type priority before the Fix Planner sees them — ensuring runbooks always outrank generic docs.
✓
Hybrid: BM25 keyword + text-embedding-3-small vectors
✓
Priority: runbook › prior_fix › docs › source_code
✓
Score threshold 0.6 — low-confidence results dropped
✓
Index grows with every resolved incident (feedback loop)

4. Security Guardrails
PII scrubbing, auth, and audit on every request
Three layers protect every request: bearer token auth on all API endpoints, PII scrubbing before any LLM call via scrub(), and an immutable audit_log row for every agent decision — ensuring full traceability with zero raw PII reaching the LLM.
✓
Bearer token auth guards all API routes (dev + prod)
✓
scrub() masks emails, IPs, UUIDs, phones before LLM call
✓
audit_log immutable row per agent decision
✓
Secrets via Azure Key Vault + Managed Identity in prod

6. Continuous Memory
Every resolved incident makes the next one faster
After a fix is merged and deployed, the Reporting Agent writes a resolution summary back to PostgreSQL. The Ingestion Worker re-indexes it into Azure AI Search as a prior_fix document. Future incidents with a matching fingerprint retrieve a proven fix on the first RAG pass.
✓
Fingerprint matching groups similar incidents automatically
✓
Merged PRs indexed as prior_fix — highest RAG priority
✓
Embedding model: text-embedding-3-small (Azure OpenAI)
✓
MTTR metrics tracked — improvement visible on dashboard
